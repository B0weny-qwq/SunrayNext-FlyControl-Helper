# 01 项目介绍

适合谁看：
- 第一次接触这个仓库的人
- 已经知道 ESP-IDF，但还不知道这个项目为什么分成两端的人

读完会得到什么：
- 知道这个项目解决什么问题
- 知道 P4 和 C5 分别负责什么
- 知道后续应该先看哪几篇文档

## 这个项目在解决什么问题

这个项目要把“网络世界”和“飞控串口世界”接起来。上位机、仿真器或地面站通常通过 UDP 或 TCP 发数据，而飞控通常通过 UART 收发数据。这个仓库里的固件负责把这两种世界连起来，并且把链路拆成两个芯片各自擅长的部分。

这里提到的飞控，一般是接收 MAVLink 消息的控制器。这里提到的 MAVLink，是一种常见的飞行器通信协议，用来传输姿态、位置、控制和状态消息。

## 为什么是两个芯片

这个项目没有把所有事情都压到一颗芯片上。它把系统分成了“主控”和“无线协处理器”两部分。

- `firmware-p4` 运行在 ESP32-P4 上，负责主逻辑
- `firmware-c5` 运行在 ESP32-C5 上，负责无线链路

这样分工的好处是很直观的。P4 更靠近应用层逻辑，所以它负责控制台命令、网络配置和 UART 桥接。C5 更靠近无线能力，所以它负责把 Wi-Fi 能力稳定地提供给 P4。

在这个项目里，C5 通过 SDIO 以从设备方式和 P4 协作。这里提到的 SDIO，可以先把它理解成“P4 和 C5 之间的一条高速板内通信通道”。

## 你会在仓库里看到什么

| 路径 | 你会在里面看到什么 |
| --- | --- |
| `firmware-p4/` | P4 侧桥接程序、控制台命令、网络和 UART 逻辑 |
| `firmware-c5/` | C5 侧 `esp-hosted` 定制固件 |
| `pcb-design/` | PCB 制造和装配资料 |
| `hardware-assets/` | 板卡图片和导出素材 |
| `docs/` | 新人上手、架构说明、接口说明和开发指南 |
